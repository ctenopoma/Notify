# SRE Alert Monitor - クライアント設定ガイド

このドキュメントでは、Windows環境で動作する「SRE Alert Monitor」デスクトップアプリの設定方法、複数Alertmanager構成（HA構成）での運用、およびネットワーク仕様について説明します。

## 1. アプリケーションの概要

このアプリケーションは、社内のPrometheus / Alertmanager基盤からのアラートを監視し、Windowsのネイティブ通知（トースト）としてユーザーに知らせる軽量な常駐アプリです。

*   **動作要件**: Windows 10 または Windows 11
*   **インストール**: 提供されるインストーラー（MSI等）を実行してください。

## 2. 基本操作とアラート確認

本アプリは起動後、タスクトレイ（画面右下の通知領域）に常駐し、バックグラウンドで動作し続けます。

*   **クイック確認（左クリック）**: トレイアイコンを**左クリック**すると、即座にメイン画面が前面にポップアップします。開いた瞬間に最新の「発火中アラート」一覧が同期されて表示されるため、通知の見逃しがないかサッと確認するのに最適です。
*   **メニュー表示（右クリック）**: アイコンを右クリックするとメニューが開き、「設定を開く」やアプリの「終了」を選択できます。

## 3. 監視設定（マルチURL対応）

設定画面（「設定」タブ）から、以下の監視オプションを変更できます。

### 複数のAlertmanagerを監視する（HA構成）
本アプリは複数のAlertmanager URLを登録し、同時にポーリング（データ取得）を行うことが可能です。

1.  **URLを追加**ボタンをクリックし、入力行を追加します。
2.  各AlertmanagerのエンドポイントURLを入力します。
    *   例: `http://10.0.0.101:9093` と `http://10.0.0.102:9093`
3.  **重複排除について**: 複数のサーバーから同じアラート（同じ `fingerprint`）が取得された場合でも、アプリ内で自動的に重複排除され、通知は1回だけ行われます。

### ポーリング間隔の即時反映
「取得間隔」スライダーで監視頻度を変更し「保存」ボタンを押すと、新しい間隔が即座に適用され、直ちに監視が再開されます。

## 4. ネットワークとプロキシ仕様 (no_proxy)

社内環境において、OSのシステムプロキシや環境変数（`HTTP_PROXY`, `HTTPS_PROXY`）が設定されている場合がありますが、本アプリは社内サーバーへの直接通信を確保するため、**すべてのプロキシ設定をバイパス（`no_proxy`）**する設計となっています。

*   社内ネットワーク内にある Alertmanager の IPアドレス（または内部DNS）へ直接HTTPリクエストを送信します。
*   プロキシサーバー経由での認証エラーや接続ブロックの影響を受けません。

## 5. エラー検知と通知仕様

### 接続エラーの可視化
いずれかのAlertmanagerへの接続が失敗した場合、アプリのヘッダー部分に赤い「Error」バッジが表示されます。マウスオーバーすると、どのURLでどのようなエラーが発生しているかの詳細を確認できます。

### サーバーダウン検知
登録されている**すべてのAlertmanager**への接続が失敗した場合、監視不能な状態（デッドマンズスイッチ状態）とみなし、WindowsのOS通知として「[CRITICAL] Connection Lost (監視サーバー接続エラー)」を発火します。

### 「解決 (Resolved)」通知
一度発火したアラートが復旧（Alertmanagerのリストから消失）した場合、アプリはそれを検知し「[RESOLVED] (アラート名)」というOS通知を送信します。これにより、インシデントの終了を手元ですぐに確認できます。
