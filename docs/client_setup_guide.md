# SRE Alert Monitor - クライアント設定ガイド

このドキュメントでは、Windows環境で動作する「SRE Alert Monitor」デスクトップアプリの設定方法、複数Grafana構成（HA構成）での運用、およびネットワーク仕様について説明します。

## 1. アプリケーションの概要

このアプリケーションは、社内の **Grafana ユニファイドアラート**（内蔵 Alertmanager）からのアラートを監視し、Windowsのネイティブ通知（トースト）としてユーザーに知らせる軽量な常駐アプリです。Grafana が Prometheus データソースに対してアラートルールを評価し、その発火中アラートをアプリがポーリングします（独立した Alertmanager は不要）。

*   **動作要件**: Windows 10 または Windows 11
*   **インストール**: 提供されるインストーラー（MSI等）を実行してください。

## 2. 基本操作とアラート確認

本アプリは起動後、タスクトレイ（画面右下の通知領域）に常駐し、バックグラウンドで動作し続けます。

*   **クイック確認（左クリック）**: トレイアイコンを**左クリック**すると、即座にメイン画面が前面にポップアップします。開いた瞬間に最新の「発火中アラート」一覧が同期されて表示されるため、通知の見逃しがないかサッと確認するのに最適です。
*   **メニュー表示（右クリック）**: アイコンを右クリックするとメニューが開き、「設定を開く」やアプリの「終了」を選択できます。

## 3. 監視設定（マルチURL対応）

設定画面（「設定」タブ）から、以下の監視オプションを変更できます。

### Grafana サービスアカウントトークンの発行
Grafana の内蔵 Alertmanager API は認証が必須です。事前に Grafana 側でトークンを発行してください。

1.  Grafana に管理者でログインし、**Administration → Users and access → Service accounts** を開く。
2.  サービスアカウントを作成（ロールは **Editor** 推奨。アラート閲覧のみなら Viewer でも可ですが、アプリからの「ミュート（サイレンス）作成」には書き込み権限が必要です）。
3.  **Add service account token** でトークンを生成し、表示された `glsa_...` 文字列をコピー。
4.  アプリの設定画面で、対象 Grafana の URL 行の**トークン欄**に貼り付けます（トークンは **URL 行ごと**に設定します。後述）。

### 複数のGrafanaを監視する（HA構成 / 複数サイト）
本アプリは複数の Grafana を登録し、同時にポーリング（データ取得）を行うことが可能です。
**Grafana ごとにサービスアカウントトークンは異なる**ため、トークンは URL と同じ行にペアで入力します。

1.  **URLを追加**ボタンをクリックし、入力行を追加します。各行は「URL ＋ そのGrafanaのトークン」の組です。
2.  各 Grafana のベース URL を入力します（パスは不要。アプリが `/api/alertmanager/grafana/api/v2/...` を補完します）。
    *   例: 1行目 `http://10.0.0.101:3000` ＋ そのサーバーの `glsa_...`、2行目 `http://10.0.0.102:3000` ＋ 別の `glsa_...`
3.  **重複排除について**: 複数のサーバーから同じアラート（同じ `fingerprint`）が取得された場合でも、アプリ内で自動的に重複排除され、通知は1回だけ行われます。

### ポーリング間隔の即時反映
「取得間隔」スライダーで監視頻度を変更し「保存」ボタンを押すと、新しい間隔が即座に適用され、直ちに監視が再開されます。

## 4. ネットワークとプロキシ仕様 (no_proxy)

社内環境において、OSのシステムプロキシや環境変数（`HTTP_PROXY`, `HTTPS_PROXY`）が設定されている場合がありますが、本アプリは社内サーバーへの直接通信を確保するため、**すべてのプロキシ設定をバイパス（`no_proxy`）**する設計となっています。

*   社内ネットワーク内にある Grafana の IPアドレス（または内部DNS）へ直接HTTPリクエストを送信します。
*   プロキシサーバー経由での認証エラーや接続ブロックの影響を受けません。

## 5. エラー検知と通知仕様

### 接続状態の判定（ハートビート方式）
本アプリは、API（Grafana 内蔵 Alertmanager の `/api/v2/alerts`）への接続に成功しただけでは「接続中」と判定しません。設定画面で指定した**テスト用アラート名**（既定値: `AlwaysFiringTest`）が実際に発火中として取得できた場合にのみ、そのサーバーを「接続中」と判定します。

これにより、API サーバー自体は応答しているものの実際のアラート評価経路（Grafana のルール評価など）に問題がある場合でも、見せかけの「監視中」表示に陥らず検知できます。各サーバーの状態（接続中 / テスト未到達 / 未接続）は設定画面のURL一覧に表示されます。テスト未到達の場合はトークンの権限不足や `AlwaysFiringTest` ルールの無効化が主な原因です。テスト用アラート名を空欄にすると、API応答のみで「接続中」と判定する従来の挙動に戻せます。

### 接続エラーの可視化
いずれかの Grafana への接続が失敗した場合、アプリのヘッダー部分に赤い「Error」バッジが表示されます。マウスオーバーすると、どのURLでどのようなエラーが発生しているかの詳細を確認できます（トークン不正時は `HTTP Error: 401` 等が表示されます）。

### サーバーダウン検知
登録されている**すべての Grafana**への接続が失敗した場合、監視不能な状態（デッドマンズスイッチ状態）とみなし、WindowsのOS通知として「[CRITICAL] Connection Lost (監視サーバー接続エラー)」を発火します。

### 「解決 (Resolved)」通知
一度発火したアラートが復旧（Grafana のアラートリストから消失）した場合、アプリはそれを検知し「[RESOLVED] (アラート名)」というOS通知を送信します。これにより、インシデントの終了を手元ですぐに確認できます。
